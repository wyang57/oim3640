<!-- Categories -->
The quiz will measure two traits. One trait focuses on how the user likes to work. The other trait focuses on the kind of tasks the user prefers. These traits will shape the profile that guides the API request.

<!-- Scoring Approach -->
Each question will add weight to one or two traits. The weights will shape the final profile. The profile will map to a small set of career themes. Each theme will connect to a keyword for the API request.

<!-- Question Style -->
The quiz will use short multiple‑choice questions. Each question will have two options. Each option will point toward a trait. This keeps the quiz simple and avoids long branching paths.

<!-- Quiz Plan -->

The quiz will shape a profile that points toward one of two traits. One trait reflects how the user approaches tasks. The other trait reflects the kind of work the user prefers. Each question will offer two choices. Each choice will guide the scoring toward one trait.

The questions will stay short so the quiz feels simple to complete. Each answer will add weight to one trait. The stronger trait will shape the keyword used in the API request.

Sample Questions

1. Do you prefer tasks that follow a steady pattern or tasks that shift during the day.
2. Do you enjoy work that involves people or work that involves tools or data.
3. Do you like planning ahead or adjusting as you go.
4. Do you feel more comfortable with structured tasks or open‑ended tasks.
5. Do you enjoy solving problems that involve logic or problems that involve communication.

The scoring will compare the two traits. The stronger trait will map to a keyword. The keyword will guide the API request and shape the occupations that appear in the results.
