<!-- # Quiz Structure Plan -->

<!-- ## Categories -->
The quiz will measure two traits. One trait focuses on how the user likes to work. The other trait focuses on the kind of tasks the user prefers. These traits will shape the profile that guides the API request.

<!-- ## Scoring Approach -->
Each question will add weight to one or two traits. The weights will shape the final profile. The profile will map to a small set of career themes. Each theme will connect to a keyword for the API request.

<!-- ## Question Style -->
The quiz will use short multiple‑choice questions. Each question will have two options. Each option will point toward a trait. This keeps the quiz simple and avoids long branching paths.
